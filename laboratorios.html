LAB 01

¿Qué es una inyección SQL?

✓ Lección aprendida

Una inyección SQL ocurre cuando una web mete texto del usuario directamente dentro de una consulta a la base de datos. Si no se protege, un atacante puede escribir código que cambia el significado de la consulta y, por ejemplo, entrar sin contraseña. Míralo en 3 minutos y luego pruébalo:

Pruébalo tú: rompe el login

Cambia entre la versión vulnerable y la segura. En la vulnerable, escribe un ataque en «Usuario» (o toca un ejemplo) y mira cómo entras sin credenciales. Luego, en la segura, comprueba que el mismo ataque ya no funciona.

Nunca concatenes texto del usuario en la consulta

            
La solución son las consultas parametrizadas (con ?): la base de datos recibe el
            texto del usuario como un dato, no como código, así que aunque escriban ' OR '1'='1
            se busca literalmente un usuario con ese nombre —y no existe—. Complétalo con validación de entradas
            y permisos mínimos en la base de datos.

          

        

      

    


    
    

      

        LAB 02
        
Monte Carlo: estimar π con azar

        ✓ Lección aprendida
      

      

        

          El método de Monte Carlo resuelve problemas… ¡lanzando dados! Usa muchos números aleatorios para
          estimar un resultado. El ejemplo clásico: si lanzas puntos al azar dentro de un cuadrado con un círculo
          dibujado, la proporción que cae dentro del círculo te permite estimar π. Míralo y pruébalo:
        

        

          

            
          

        


        
Lánzalo tú: ¿cuánto vale π?

        
Cada punto azul cae dentro del círculo; cada punto rojo, fuera. Lanza miles y observa cómo el cálculo 4 × (dentro ÷ total) se acerca al valor real de π (3.1416…).

Cuantos más puntos lances, más se acerca la estimación a π. Eso es Monte Carlo: el azar, a gran escala, da respuestas precisas.
