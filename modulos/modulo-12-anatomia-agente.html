Nivel 04 · Agentes de IA

La anatomía de un agente

Un chatbot responde; un agente actúa. La diferencia son tres piezas —prompt de sistema, memoria y herramientas— y un bucle de razonamiento. Cuando las entiendas, sabrás diseñar cualquier agente, no solo usarlo.

35 minutos Nivel intermedio La base de todo agente 5 secciones + repaso
De responder a actuar

01 Chatbot vs. agente

Un chatbot conversa: le preguntas, te responde. Un agente además hace cosas: busca información, llama a una API, crea una tarea en Workfront, envía un correo. No se queda en palabras; ejecuta acciones para cumplir un objetivo.

La clave es que el agente puede usar herramientas y decidir cuáles usar y en qué orden, apoyándose en lo que recuerda. Eso lo acerca a un colaborador digital.

La idea clave

Agente = modelo + objetivo + herramientas

Le das un cerebro (el modelo), una misión (el objetivo) y manos (las herramientas). El agente razona sobre cómo usar esas manos para cumplir la misión.

03 Las tres piezas

Todo agente, por sofisticado que sea, se compone de estas tres partes:

Prompt de sistemaLa personalidad y las reglas

  • Quién es y cómo se comporta
  • Qué puede y qué no puede hacer
  • El tono y los límites

MemoriaLo que recuerda

  • La conversación actual
  • Datos del usuario e historial
  • A menudo una base de datos

HerramientasLo que puede hacer

  • Buscar, calcular, enviar
  • APIs (Workfront, correo…)
  • Cada una amplía sus poderes
Recuerda

La memoria vive en una base de datos

Aquí reaparece el Nivel 02: para recordar entre conversaciones, el agente guarda su memoria en una base de datos relacional. Sin ella, empieza de cero cada vez.

04 El bucle del agente

Un agente no responde de golpe: razona en pasos. Piensa, usa una herramienta, observa el resultado y decide el siguiente paso, hasta cumplir el objetivo. Así se ve por dentro:

bucle-agente.txt
# Objetivo: "¿Cuántas tareas abiertas tiene el proyecto Acme?"

1. PENSAR    Necesito consultar Workfront.
2. ACTUAR    Herramienta → API de Workfront (proyecto = Acme)
3. OBSERVAR  Respuesta: { "tareas_abiertas": 7 }
4. PENSAR    Ya tengo el dato. Puedo responder.
5. RESPONDER "El proyecto Acme tiene 7 tareas abiertas."

# Si hubiera necesitado más datos, repetiría el bucle
# (pensar → actuar → observar) antes de responder.

05 Ponle límites claros

Un agente con herramientas puede hacer daño si no se controla. Diséñalo con frenos desde el inicio:

  • Permisos mínimos: dale acceso solo a lo que necesita, no a todo.
  • Confirmación humana: para acciones sensibles (borrar, pagar, enviar a clientes).
  • Instrucciones claras: en el prompt de sistema, di qué NO debe hacer.
  • Registro: guarda qué hizo y cuándo, para poder auditarlo.
Adelanto

La gobernanza es un nivel entero

Permisos, privacidad y uso responsable son tan importantes que les dedicamos el Nivel 05 completo. Un agente potente sin gobierno es un riesgo, no una ventaja.

06 Comprueba lo aprendido

1. ¿Qué distingue a un agente de un chatbot normal?

Correcto

El agente actúa: usa herramientas y decide cómo cumplir la misión, no solo conversa.

2. ¿Cuáles son las tres piezas de un agente?

Correcto

Personalidad/reglas (prompt de sistema), lo que recuerda (memoria) y lo que puede hacer (herramientas).
