Nivel 05 · Skills y Gobernanza

¿Qué es una Skill?

Una skill es una habilidad extra que le enseñas a la IA: un manual con instrucciones que el modelo activa solo cuando lo necesita. Aquí verás qué son los archivos .md, cómo se estructura una skill y cómo crear la tuya.

35 minutos Nivel avanzado Amplía lo que la IA sabe hacer 5 secciones + repaso
Una habilidad que le enseñas a la IA

01 Una habilidad bajo demanda

Una skill es un paquete de instrucciones y recursos que le da a la IA una capacidad concreta: «redactar informes con nuestro formato», «revisar un contrato», «crear una tarea en Workfront». En lugar de explicárselo cada vez, lo guardas una vez como skill y la IA lo usa cuando hace falta.

Lo elegante es que el modelo solo la carga cuando es relevante. Tú describes «para qué sirve» y «cuándo usarla»; la IA la activa sola en el momento adecuado, sin saturar la conversación.

La idea clave

Es como contratar a un especialista

Sin skills, la IA es un generalista. Con skills, es como tener a mano especialistas que aparecen justo cuando su tema entra en juego, cada uno con su manual de instrucciones.

03 Los archivos .md (Markdown)

Las skills se escriben en Markdown, archivos con extensión .md. Markdown es texto normal con unos pocos símbolos para dar formato: # para títulos, - para listas, **negrita**. Legible para humanos y para la IA.

Por eso es el formato del ecosistema: los README.md de GitHub, la documentación y las skills, todo es Markdown.

ejemplo.md
# Título principal
## Subtítulo

Texto normal con una palabra en **negrita**.

- Punto de una lista
- Otro punto

> Una cita o nota destacada

04 Anatomía de una skill

Una skill suele ser una carpeta con un archivo SKILL.md. Tiene dos partes: una ficha con datos y las instrucciones.

SKILL.md
---
name: reporte-semanal
description: Genera el reporte semanal de un proyecto
  de Workfront con nuestro formato. Usar cuando pidan
  "reporte semanal" o "resumen del proyecto".
---

# Reporte semanal

## Pasos
1. Obtener las tareas del proyecto en Workfront.
2. Resumir avances y bloqueos.
3. Formatear con: Título, Logros, Riesgos, Próximos pasos.
Lo más importante

La descripción decide cuándo se activa

El campo description es clave: le dice a la IA para qué sirve y cuándo usarla. Una buena descripción hace que la skill aparezca en el momento justo; una vaga, que nunca se use.

05 Cómo crear tu propia skill

  • Elige una tarea repetitiva que expliques a la IA una y otra vez.
  • Escribe el SKILL.md: nombre, una descripción clara de cuándo usarla, y los pasos.
  • Añade recursos si hace falta: plantillas, ejemplos, un archivo con el formato.
  • Pruébala: pide algo que debería activarla y ajusta la descripción hasta que aparezca sola.
  • Compártela: guárdala en GitHub (Módulo 16) para que la use todo el equipo.
Consejo

La IA te ayuda a escribir la skill

No partas de cero: pídele a un chatbot «ayúdame a escribir una skill que haga X, con su descripción y sus pasos». Tú aportas el conocimiento del proceso; la IA lo ordena en formato.
