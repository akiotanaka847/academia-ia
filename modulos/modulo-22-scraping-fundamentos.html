Nivel 06 · Web Scraping

Fundamentos del web scraping

El web scraping es extraer datos de páginas web de forma automática. Bien usado, ahorra horas de copiar y pegar; mal usado, cruza líneas legales. Empezamos por lo más importante: qué es, cuándo se puede, y cómo funciona.

Copiar y pegar, automatizado

01 ¿Qué es el web scraping?

Web scraping es usar un programa para leer una página web y quedarse con los datos que te interesan —precios, contactos, artículos— de forma automática y estructurada. Piénsalo como un «copiar y pegar» a gran velocidad, que además ordena lo que recoge en una tabla o en JSON.

Aquí reaparece el Nivel 02: como una web es HTML, el scraping consiste en localizar los datos dentro de esa estructura y extraerlos.

Casos legítimos

Para qué se usa en la empresa

Monitorizar precios de la competencia, reunir estadísticas públicas, recopilar noticias del sector, alimentar una base de datos interna. El objetivo es ahorrar trabajo manual sobre datos que ya son accesibles.

03 ¿Es legal? Lee esto primero

El scraping no es ilegal en sí, pero cómo y qué extraes marca la diferencia entre una práctica legítima y un problema serio. Antes de scrapear nada, revisa estas cuatro reglas:

Datos públicosSí, con cuidado

  • Información visible sin login
  • No datos privados ni tras registro

robots.txtRespétalo

  • El archivo que dice qué se puede rastrear
  • Míralo antes de empezar

Términos de servicioLéelos

  • Algunas webs lo prohíben
  • Ignorarlos tiene consecuencias

Datos personalesZona roja

  • Nombres, correos, teléfonos
  • Sujeto a RGPD y privacidad
Regla de oro

Ante la duda, no lo hagas (o consúltalo)

No sobrecargues los servidores de nadie, no extraigas datos personales sin base legal, y respeta lo que la web pide en su robots.txt y sus términos. Un scraping agresivo o de datos protegidos puede ser una infracción. Ante casos sensibles, consulta con el área legal.

04 Cómo funciona, en tres pasos

scraping-basico.txt
1. PEDIR       Descargar el HTML de la página
2. LOCALIZAR  Encontrar el dato dentro del HTML
             (ej. el precio está en <span class="precio">)
3. EXTRAER    Guardar el valor en una tabla o JSON

# Repetir para cada página o producto

Para hacerlo tienes tres caminos, de menos a más técnico:

  • Sin código: extensiones y apps (Instant Data Scraper, Octoparse) que señalas y extraen.
  • Con IA: pegas el contenido o la URL y pides a la IA que estructure los datos.
  • Con código: Python (con librerías como BeautifulSoup) para casos a medida y a escala.

05 ¿Scraping o API?

Antes de scrapear, pregunta si la web ofrece una API (Nivel 02). Si existe, casi siempre es mejor: es la vía oficial, más estable y pensada para eso. El scraping es el plan para cuando no hay API disponible.

Orden recomendado

1) API oficial → 2) API oculta → 3) scraping de HTML

Primero busca una API pública. Si no hay, en el Módulo 24 verás cómo descubrir «APIs ocultas». Y solo si nada de eso existe, scrapeas el HTML directamente. De lo más limpio a lo más frágil.
