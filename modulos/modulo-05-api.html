Nivel 02 · Conceptos técnicos

¿Qué es una API?

Es la palabra que más se repite cuando hablamos de automatización e integraciones. Aquí la entenderás para siempre con una analogía sencilla, verás cómo funciona una petición y por qué las APIs son el pegamento que conecta todas tus herramientas.

25 minutos Nivel básico La base de la automatización 5 secciones + repaso
El pegamento entre programas

01 La analogía del restaurante

API significa Application Programming Interface. Suena técnico, pero la idea es simple: es un intermediario que permite que dos programas se hablen sin conocer sus interioridades.

Imagina un restaurante. Tú (una aplicación) no entras a la cocina (otra aplicación) a cocinar. Le dices al mesero lo que quieres; él lo lleva a la cocina y te trae el plato. La API es ese mesero: recibe tu pedido, lo entrega y te devuelve la respuesta.

La idea clave

No necesitas saber cómo funciona por dentro

Cuando tu app pide el clima a una API, no le importa cómo mide la temperatura ese servicio. Solo hace la pregunta correcta y recibe la respuesta. Esa separación es lo que hace posible conectar todo.

03 Petición y respuesta

Toda API funciona con dos movimientos: tú envías una petición (request) y recibes una respuesta (response).

EndpointLa dirección

  • La URL a la que llamas
  • Ej.: /clima/madrid
  • Cada uno hace una cosa

MétodoLa intención

  • GET: pedir datos
  • POST: enviar datos
  • PUT / DELETE: actualizar / borrar

RespuestaEl resultado

  • Suele venir en JSON
  • Trae un código (200 = ok)
  • 404 = no encontrado

Así se ve una petición real para pedir el clima y su respuesta:

peticion-api.txt
# Petición: método + endpoint
GET https://api.clima.com/v1/madrid

# Respuesta (en JSON):
{
  "ciudad": "Madrid",
  "temperatura": 28,
  "estado": "soleado"
}

04 Por qué deberían importarte

Las APIs son la razón por la que puedes automatizar de verdad. Sin ellas, cada herramienta sería una isla.

  • Conectan tus herramientas: que un formulario cree una tarea en Workfront automáticamente.
  • Dan poderes a la IA: un agente usa APIs para consultar datos, enviar correos o publicar.
  • Ahorran trabajo manual: nada de copiar y pegar entre sistemas.
  • Están en todo: pagos, mapas, mensajería, clima… casi todo servicio ofrece una API.

Adobe Workfront tiene API

05 Glosario mínimo

  • API key: una contraseña que identifica quién hace la petición (no la compartas).
  • Rate limit: cuántas peticiones puedes hacer por minuto antes de que te frene.
  • Documentación: el «manual» de una API; siempre empieza por leerla.
  • Webhook: al revés que una API normal —el servicio te avisa a ti cuando pasa algo.

Correcto

Correcto
