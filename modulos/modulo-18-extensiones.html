Nivel 05 · Skills y Gobernanza

Extensiones y capacidades

Las skills son una forma de ampliar la IA, pero no la única. Aquí ordenamos todas las maneras de darle superpoderes —herramientas, conectores, extensiones— para que sepas cuál usar en cada caso y cómo se combinan.

Más formas de ampliar la IA

01 Un modelo con accesorios

Un modelo de lenguaje «pelado» solo sabe conversar. Para que sea útil de verdad, le añadimos capacidades: acceso a datos, acciones sobre otras apps, conocimiento especializado. Hay varios tipos de accesorios y conviene no confundirlos.

La idea clave

Skills = saber; herramientas = poder

Simplificando: una skill le enseña cómo hacer algo (conocimiento); una herramienta le da la capacidad de ejecutar una acción (poder). Las mejores soluciones combinan ambas.

03 Los cuatro tipos de capacidad

SkillsConocimiento

  • Instrucciones en .md
  • «Cómo hacer X»
  • Se activan solas

HerramientasAcciones

  • Funciones que la IA invoca
  • Buscar, calcular, enviar
  • «Poder hacer X»

Conectores (MCP)Acceso a apps

  • Estándar para conectar sistemas
  • Workfront, Drive, bases de datos
  • Datos y acciones en vivo

ExtensionesEn tus apps

  • IA dentro del navegador o editor
  • Actúan sobre lo que ves
  • Ej.: asistentes en el IDE
MCP, en una frase

El «USB» de la IA

MCP (Model Context Protocol) es un estándar para enchufar apps y datos a un modelo, igual que el USB conecta cualquier dispositivo. Un conector MCP de Workfront deja que la IA lea y actúe en él.

04 Cómo combinarlas

En un caso real se mezclan. Un agente que prepara reportes de Workfront usaría:

  • Una skill con el formato y los pasos del reporte (el «cómo»).
  • Un conector (MCP) a Workfront para leer los datos reales (el «acceso»).
  • Una herramienta de envío para mandar el reporte por correo (la «acción»).
Regla práctica

Añade solo lo que necesitas

Cada capacidad extra es más poder… y más superficie de riesgo. Da a cada agente únicamente las herramientas y conexiones que su tarea requiere. Enlaza directo con la gobernanza (Módulo 21).

05 Comprueba lo aprendido

1. ¿Cuál es la diferencia entre una skill y una herramienta?

Correcto

Saber (skill) frente a poder (herramienta). Las mejores soluciones combinan ambas.

2. ¿Qué es MCP?

Correcto

MCP (Model Context Protocol) enchufa sistemas como Workfront a la IA de forma estándar.
