Nivel 05 · Skills y Gobernanza

Por qué necesitas GitHub

Antes de entrar en skills y en el ecosistema de la IA, hay una herramienta que abre todas esas puertas: GitHub. Es donde vive el código del mundo —y casi todas las skills, plugins y agentes que querrás usar o compartir. Tener una cuenta es el primer paso.

25 minutos Nivel avanzado La base del ecosistema 5 secciones + repaso
Donde vive el código del mundo

01 ¿Qué es GitHub?

GitHub es la plataforma donde se guarda, versiona y comparte código (y cualquier archivo de texto: documentos, configuraciones, skills). Piénsalo como un Google Drive con máquina del tiempo: cada cambio queda registrado, puedes volver atrás y varias personas colaboran sin pisarse.

Cada proyecto vive en un repositorio (o «repo»): una carpeta con todo su historial. Es el estándar mundial: si una herramienta de IA es abierta, casi seguro está en GitHub.

La idea clave

No es solo para programadores

Guardar prompts, plantillas, documentación o la configuración de tus agentes también es trabajo que conviene versionar y respaldar. GitHub sirve para cualquier archivo de texto, no solo código.

03 Por qué te conviene tener cuenta

Una cuenta de GitHub (gratuita) te desbloquea cinco cosas muy útiles para trabajar con IA:

  • Respaldas y versionas tu trabajo: nunca más un «se me borró» o un «¿cuál era la última versión?».
  • Publicas webs gratis: con GitHub Pages pones online un sitio —como esta academia— en minutos.
  • Accedes al ecosistema de IA: la mayoría de skills, plugins, agentes y herramientas abiertas se comparten aquí.
  • Compartes lo que creas: tus propias skills o plantillas, con tu equipo o con el mundo.
  • Colaboras en orden: varias personas sobre el mismo proyecto, con historial de quién cambió qué.

04 Cuatro palabras que oirás

RepositorioEl proyecto

  • La carpeta con todo el historial
  • Público o privado
  • «El repo del proyecto X»

CommitUna foto guardada

  • Un punto guardado en el tiempo
  • Con un mensaje que lo describe
  • Puedes volver a él

README.mdLa portada

  • Un archivo .md
  • Explica de qué va el repo
  • Lo primero que se lee

PagesPublicar

  • Web gratis desde un repo
  • Ideal para sitios estáticos
  • Una URL para compartir

05 Cómo empezar

  • Crea tu cuenta gratis en github.com con tu correo de trabajo.
  • Activa la verificación en dos pasos (2FA): es tu identidad de desarrollador, protégela.
  • Explora: busca «claude skills», «n8n templates» o tu herramienta favorita y mira sus repos.
  • Crea tu primer repo: aunque sea para guardar tus prompts y plantillas del equipo.
Para la empresa

Una organización para el equipo

GitHub permite crear una «organización» donde el equipo comparte repos privados con permisos por persona. Es la forma ordenada de guardar el conocimiento técnico de la empresa —enlaza con la gobernanza que veremos al final de este nivel.

06 Comprueba lo aprendido

1. ¿Qué es un «repositorio» en GitHub?

Correcto

El repo guarda el proyecto completo y su historial; puede ser público o privado.

2. ¿Por qué le conviene una cuenta de GitHub a alguien que trabaja con IA?

Correcto

Es la base del ecosistema: descubrir, compartir, versionar y publicar. No hace falta ser programador.
