Nivel 02 · Conceptos técnicos

Bases de datos relacionales

Cierra el Nivel 02. Una base de datos es la memoria ordenada de cualquier sistema. Vas a entender tablas, relaciones y una consulta básica —lo justo para saber dónde «vive» la información de tus agentes.

25 minutos Nivel básico La memoria de los sistemas 4 secciones + repaso
La memoria ordenada

01 Una hoja de cálculo con superpoderes

Una base de datos relacional guarda información en tablas, igual que una hoja de Excel: filas (cada registro) y columnas (cada atributo). La diferencia es que puede manejar millones de registros y relacionar unas tablas con otras de forma fiable.

Por ejemplo, una tabla de Clientes:

tabla · clientes
id  | nombre       | ciudad     | vip
1   | Ana Torres   | Madrid     | true
2   | Luis Gómez   | Sevilla    | false
3   | Marta Ruiz   | Valencia   | true
La clave primaria

Cada fila tiene un identificador único

La columna id es la clave primaria: un número único que identifica a cada registro sin confusión, aunque dos clientes se llamen igual.

03 Relaciones y consultas

Lo «relacional» viene de conectar tablas. Una tabla de Pedidos no repite los datos del cliente: solo guarda su id. Así, si Ana cambia de ciudad, se actualiza en un único sitio.

Para pedir datos usamos SQL, un lenguaje que se lee casi como inglés:

consulta.sql
SELECT nombre, ciudad
FROM clientes
WHERE vip = true;

-- Devuelve: Ana Torres (Madrid), Marta Ruiz (Valencia)
Cómo leerlo

SELECT … FROM … WHERE …

SELECT qué columnas quieres, FROM de qué tabla, WHERE con qué condición. Con ese patrón entiendes la mayoría de consultas que verás.

04 Por qué tu agente la necesita

Un agente de IA sin base de datos olvida todo entre conversaciones. Con ella, gana memoria:

  • Recordar: quién es cada usuario, sus preferencias, su historial.
  • Registrar: cada pedido, ticket o interacción para consultarlo después.
  • Consultar: responder «¿cuántos pedidos hizo este cliente?» con datos reales.
  • Auditar: dejar rastro de lo que el agente hizo y cuándo.

05 Comprueba lo aprendido

1. ¿Para qué sirve la clave primaria (columna id)?

Correcto

El id único evita confundir registros aunque dos tengan el mismo nombre.

2. En SQL, ¿qué parte indica la condición del filtro?

Correcto

WHERE define la condición: «solo las filas donde vip = true».
